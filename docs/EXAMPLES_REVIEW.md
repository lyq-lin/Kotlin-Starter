# 代码示例补全结果（除 01 外）

本轮已按“注释解释 + 技术覆盖”标准，对 `02` 到 `15` 的入口示例进行了补全。

## 补全原则

1. 每个示例都增加“为什么这样写”的说明注释。
2. 每个主题补齐至少 1~2 个关键延展点，而不只保留最小可运行代码。
3. 输出内容尽量能体现新增语法点的实际效果。

## 模块补全清单

- `02-VariablesAndTypes`：补充 `val/var` 差异、`const val`、可空类型、`lateinit`、`lazy`。
- `03-Modifiers`：补充 `internal`，并明确 `open/protected/private/override` 场景。
- `04-Functions`：补充具名参数与 `Unit` 返回函数。
- `05-ControlFlow`：补充 `step/downTo` 与 `when` 表达式结果。
- `06-ClassesAndDataClasses`：补充 `copy`、解构声明。
- `07-Collections`：补充 `map/filter/fold` 的组合用法。
- `08-NullSafety`：补充 `let` 与早返回写法。
- `09-Extensions`：补充扩展函数限制说明（非真实成员）。
- `10-EnumsSealedResult`：补充 sealed + `when` 穷尽分支示例。
- `11-LambdaGenericsScopes`：补充 `also` 与作用域函数职责说明。
- `12-PracticalScenarios`：改用 `BigDecimal`，规避金额精度问题。
- `13-ImportantPatterns`：补充 `Result` 的 `mapCatching/recover` 链式处理。
- `14-ai-cli`：补充入口职责注释（启动与解耦）。
- `15-web-api-demo`：补充 `Content-Type`、UTF-8、404 fallback 路由示例。
